Codex Web Term PM2 switch steps
================================

Run these commands in your own local PowerShell window on the PC.
Do not run them inside the web terminal.

1. Go to the project folder

cd <your-project-folder>\codex-web-term

2. Confirm pm2 is installed

npm ls pm2

3. Stop the current server on port 3210

$listener = Get-NetTCPConnection -LocalPort 3210 -State Listen -ErrorAction SilentlyContinue | Select-Object -First 1 -ExpandProperty OwningProcess
if ($listener) { Stop-Process -Id $listener -Force }

4. Start the production PM2-managed server

.\node_modules\.bin\pm2.cmd start ecosystem.config.cjs --only codex-web-term --update-env

5. Check PM2 status

.\node_modules\.bin\pm2.cmd status

Expected:
- process name: codex-web-term
- status: online

6. View logs

.\node_modules\.bin\pm2.cmd logs codex-web-term --lines 80

Expected log output should include:
Codex Web Term listening on http://localhost:3210

7. Verify health endpoint

Invoke-WebRequest -UseBasicParsing http://127.0.0.1:3210/api/health | Select-Object -ExpandProperty Content

Expected:
- JSON response
- contains "ok":true

Common commands after switching
-------------------------------

Restart production mode:

cd <your-project-folder>\codex-web-term
.\scripts\restart.ps1 -Mode prod

Check status:

cd <your-project-folder>\codex-web-term
.\scripts\status.ps1

View logs:

cd <your-project-folder>\codex-web-term
.\scripts\logs.ps1

Development watch mode:

cd <your-project-folder>\codex-web-term
.\scripts\start.ps1 -Mode dev

Production managed mode:

cd <your-project-folder>\codex-web-term
.\scripts\start.ps1 -Mode prod
